## 显性时间消耗

这就是今天的往返记录

| 起点           | 终点           | 方式     | 时间     |
| -------------- | -------------- | -------- | -------- |
| 靓园           | 黄龙体育中心   | 共享单车 | 五分钟   |
| 黄龙体育中心   | 浙大紫金港站   | 地铁     | 四十分钟 |
| 浙大紫金港站   | 东教           | 共享单车 | 十分钟   |
| 东教           | 紫金港路公交站 | 共享单车 | 五分钟   |
| 紫金港路公交站 | 玉泉公交站     | 公交车   | 六十分钟 |
| 玉泉公交站     | 宿舍           | 共享单车 | 十分钟   |

而如果是以往骑自行车呢

| 起点 | 终点 | 方式   | 时间     |
| ---- | ---- | ------ | -------- |
| 靓园 | 东教 | 自行车 | 三十分钟 |
| 东教 | 宿舍 | 自行车 | 三十分钟 |

可以很明显看到，在这不长不短的八九公里(单程)中，骑车反而是最好的选择，消耗时间最短，而且时间消耗期望稳定，不会出现很严重的延缓

## 隐性时间消耗

如果你同时有两个支付宝账号，同时想要使用乘车码，不好意思，这是不可能的，你需要频繁的切换账号以满足自己的需求，而支付宝极其糟糕的体验让你每次都不得不花费数十秒完成这个操作，遇到网络不稳定的时候更是抓狂(地铁站)

如果想要一劳永逸，将卡解绑，所有的都绑定到一个账号，你不得不需要停止使用他们的服务长达3-7天，以完成这个操作，而且整个解绑过程完全不像领取过程那么方便，各种按钮都隐藏的很深(基于他们糟糕的 UI/UX 设计和丑陋的 H5 开发)，领卡一瞬间，退卡三五天。

同样的事情还会出现在其他的方面，比如电子笔记，电子书，虽然有可能单位时间的效率会提高(有时候使用高级的东西甚至不会提高效率)，但总体工作成果却是不如简单粗暴的直接完成的，应该是因人而异吧
